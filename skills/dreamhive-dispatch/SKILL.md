# 集群调度 — 智能技能路由

你可以访问一个名为 DreamHive 的智能技能调度系统。
它索引所有已安装的技能，并为任何任务自动选择最佳技能。

## 使用场景

在以下情况激活：
- 用户的任务比较复杂，可能受益于专业技能
- 你不确定该用哪个技能
- 任务涉及多个步骤，不同技能可以分别处理
- 用户说 "用集群" 或 "自动调度"

## 如何调度

### 第一步：分析任务

将用户的请求拆分为子任务。对每个子任务识别：
- 需要什么能力（如 "代码审查"、"调试"、"git 操作"）
- 期望什么输出

### 第二步：查询集群

为每个子任务运行推荐引擎：

```bash
python3 "${CLAUDE_PLUGIN_ROOT}/scripts/dreamhive.py" suggest "<子任务描述>"
```

### 第三步：选择并执行

对每个推荐的技能：
1. 使用 Skill 工具调用技能: `技能名`
2. 记录调用结果:
   ```bash
   python3 "${CLAUDE_PLUGIN_ROOT}/scripts/dreamhive.py" invoke "<技能名>" ok "<简要上下文>"
   ```
3. 如果技能不适用或失败:
   ```bash
   python3 "${CLAUDE_PLUGIN_ROOT}/scripts/dreamhive.py" invoke "<技能名>" fail "<原因>"
   ```

### 第四步：串联输出

需要串联技能时：
- 将技能 A 的输出作为上下文传递给技能 B
- 使用以下模式:
  1. 执行技能 A，提取关键发现
  2. 调用技能 B 时包含: "基于 [技能 A 的输出]: <发现>"
  3. 记录两次调用

### 第五步：报告

调度完成后，提供摘要:
```
🐝 集群调度报告
━━━━━━━━━━━━━━━━━━━━━━
已使用技能: skill-a → skill-b → skill-c
结果: ✓ skill-a → ✓ skill-b → ✓ skill-c
```

## 调度规则

1. **每个任务最多 3 个技能** — 不要过度调度。如果 >3 个技能相关，
   按得分选 Top-3。
2. **快速失败** — 如果技能读取后不适用，跳过它，继续下一个。
   不要强行匹配。
3. **记录一切** — 每次调用（成功或失败）都能改善未来的推荐。
   始终运行 invoke 命令。
4. **尊重用户选择** — 如果用户覆盖了你的建议，遵循用户的选择
   并记录下来。
